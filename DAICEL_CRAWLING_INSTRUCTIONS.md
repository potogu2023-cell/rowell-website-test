## 📊 数据提取要求

### 1. 产品描述（description）

**优先级**: ⭐⭐⭐⭐⭐ 最高

**提取目标**:
- 产品的详细描述文本
- 应用场景和特性说明
- 技术优势和性能特点

**质量标准**:
- 优先提取官网的产品描述段落
- 保留完整的句子和段落
- 去除HTML标签和多余空格
- 最小长度：50字符

**描述质量等级**（descriptionQuality）:
- `high`: 详细描述，>200字符，包含应用场景和技术特点
- `medium`: 中等描述，100-200字符，包含基本产品信息
- `low`: 简短描述，50-100字符，仅基本信息
- `extracted`: 从目录号或规格中提取的描述，<50字符
- `none`: 无描述

### 2. 技术规格（specifications）

**优先级**: ⭐⭐⭐⭐⭐ 最高

**提取目标**:
- 所有可见的技术参数
- 产品规格表中的数据
- 性能指标

**常见规格字段**（Daicel产品）:

**手性色谱柱特有字段**:
- Chiral Selector（手性选择剂）
- Particle Size（粒径）
- Pore Size（孔径）
- Column Length（柱长）
- Inner Diameter（内径）
- pH Range（pH范围）
- Max Pressure（最大压力）
- Max Temperature（最高温度）
- USP Classification（USP分类）
- End-Capping（封端）
- Carbon Load（碳载量）
- Surface Area（比表面积）

**通用字段**:
- Product Code（产品代码）
- Catalog Number（目录号）
- Material（材质）
- Compatibility（兼容性）
- Application（应用）

**JSON格式示例**:
```json
{
  "Chiral Selector": "Amylose tris(3,5-dimethylphenylcarbamate)",
  "Particle Size": "3 µm",
  "Pore Size": "100 Å",
  "Column Length": "150 mm",
  "Inner Diameter": "4.6 mm",
  "pH Range": "2.0-7.5",
  "Max Pressure": "200 bar",
  "Max Temperature": "40°C",
  "USP Classification": "L95",
  "End-Capping": "Yes",
  "Carbon Load": "15%",
  "Surface Area": "300 m²/g"
}
```

**质量标准**:
- 至少提取3个规格字段
- 保留原始单位（mm、µm、Å等）
- 数值和单位之间保留空格
- 字段名使用英文

## 🔧 技术要求

### 1. 爬虫程序要求

**推荐工具**:
- Python + Selenium（动态页面）
- Python + BeautifulSoup（静态页面）
- Node.js + Puppeteer

**必需功能**:
- 支持动态页面加载
- 支持JavaScript渲染
- 错误重试机制（最多3次）
- 日志记录

### 2. 数据提取规则

**描述提取**:
1. 优先提取产品页面的主要描述段落
2. 如果没有描述，提取产品特性列表
3. 如果仍然没有，从目录号生成简单描述

**规格提取**:
1. 查找规格表格（table）
2. 查找规格列表（dl、ul）
3. 查找规格段落（包含"Specifications"、"Technical Data"等关键词）
4. 提取所有键值对，转换为JSON格式

### 3. 错误处理

**常见错误及处理**:
- 页面404：标记为失败，记录日志
- 页面超时：重试3次，仍失败则跳过
- 数据提取失败：记录警告，继续下一个产品
- 网络错误：等待10秒后重试

### 4. 性能要求

- 单个产品爬取时间：<5秒
- 总执行时间：<90分钟（277个产品）
- 并发请求：建议1-2个（避免被封IP）
- 请求间隔：建议2-3秒

## 🚨 风险和应急预案

### 风险识别

| 风险 | 可能性 | 影响 | 应对策略 |
|------|--------|------|---------|
| 官网结构变化 | 中 | 高 | 调整选择器，重新测试 |

**情况1: 测试阶段失败率>30%**
- 停止全量爬取
- 分析失败原因
- 优化爬虫程序
- 重新测试

**情况2: 全量爬取中断**
- 记录已完成的产品
- 从中断点继续
- 合并结果文件

**情况3: 数据质量不达标**
- 识别问题产品
- 针对性重新爬取
- 补充缺失数据
