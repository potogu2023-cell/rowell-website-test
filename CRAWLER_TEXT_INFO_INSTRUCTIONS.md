## 🎯 当前任务目标

### 必需字段 (Must Have)

#### 1. 完整产品名称 (name) - 🔴 最高优先级

**当前问题**: 部分产品名称被截断或不完整

**要求**:
- 完整的产品名称，不能被截断
- 包含所有型号、规格信息
- 保持品牌官网的原始格式

**示例**:
```
❌ 错误: "Thermo Fisher Scientific|Chromatography Supply Gua..."
✅ 正确: "Thermo Fisher Scientific Chromatography Supply Guard Column, C18, 4.6mm x 10mm"
```

#### 2. 产品描述 (description) - 🔴 最高优先级

**当前状态**: 76.9%产品缺失描述

**要求**:
- 完整的产品描述文本（100-500字）
- 包含产品用途、特点、应用场景
- 使用纯文本格式，移除HTML标签
- 保持专业术语的准确性

**示例**:
```
"这款HPLC色谱柱采用C18固定相，专为反相色谱分离设计。
适用于药物分析、环境检测和食品安全等领域。
具有优异的峰形、高分离度和长使用寿命等特点。
兼容各种HPLC系统，操作压力可达400 bar。"
```

#### 3. 技术规格 (specifications) - 🔴 最高优先级

**当前状态**: 100%产品缺失规格

**要求**:
- 结构化的技术参数（JSON格式）
- 包含所有关键规格信息
- 统一单位格式
- 确保数据准确性

**JSON格式示例**:
```json
{
  "固定相": "C18",
  "粒径": "5 μm",
  "孔径": "100 Å",
  "柱长": "250 mm",
  "内径": "4.6 mm",
  "pH范围": "2-8",
  "最大压力": "400 bar",
  "温度范围": "10-60°C",
  "材质": "不锈钢",
  "端接": "标准1/4-28螺纹"
}
```

**常见规格字段**（根据产品类型选择）:

**色谱柱类产品**:
- 固定相/填料类型
- 粒径 (particle size)
- 孔径 (pore size)
- 柱长 (column length)
- 内径 (inner diameter)
- pH范围
- 最大压力
- 温度范围

**样品瓶类产品**:
- 容量 (volume)
- 材质 (material)
- 颜色 (color)
- 瓶口类型 (cap type)
- 尺寸 (dimensions)
- 包装数量 (pack size)

**SPE柱类产品**:
- 填料类型
- 填料质量
- 管体积
- 床体积
- 应用类型

#### 4. 详细描述 (detailedDescription) - 🟡 可选

**要求**:
- 更详细的技术说明
- 应用案例
- 使用建议
- 可以包含HTML格式

## 📝 数据格式要求

### CSV格式（推荐）

```csv
productId,partNumber,brand,name,description,specifications,detailedDescription
AGIL-51832068,5183-2068,Agilent,"2mL Crimp Vial Amber Write-On","Amber glass vial for...","{""volume"":""2mL"",""material"":""glass""}","Detailed description..."
```

### 字段说明

| 字段名 | 类型 | 必需 | 说明 | 最大长度 |
|--------|------|------|------|---------|
| productId | String | ✅ | 产品ID（用于匹配） | 128 |
| partNumber | String | ✅ | 零件号（用于匹配） | 128 |
| brand | String | ✅ | 品牌名称 | 64 |
| name | String | ✅ | 完整产品名称 | 无限制 |
| description | String | ✅ | 产品描述 | 无限制 |
| specifications | JSON | ✅ | 技术规格（JSON格式） | 无限制 |
| detailedDescription | String | ⚠️ | 详细描述（可选） | 无限制 |

### 重要说明

1. **不要包含imageUrl字段** - 图片将在后续阶段处理
2. **specifications必须是有效的JSON** - 可以使用在线JSON验证工具检查
3. **使用UTF-8编码** - 确保中文和特殊字符正确显示
4. **统一单位格式** - 例如: "5 μm", "250 mm", "4.6 mm"

## 🔍 爬取策略建议

### 推荐方法：基于零件号精确匹配

1. **使用零件号搜索**
   ```
   在品牌官网搜索框输入: {partNumber}
   例如: 5183-2068
   ```

2. **进入产品详情页**
   - 找到完全匹配的产品
   - 验证产品名称是否一致

3. **提取文字信息**
   - 产品名称：通常在页面标题或H1标签
   - 产品描述：通常在"Overview"或"Description"部分
   - 技术规格：通常在"Specifications"或"Technical Data"表格中

4. **数据清洗**
   - 移除HTML标签
   - 统一单位格式
   - 转换规格为JSON格式

### 技术规格提取示例

**网页上的表格**:
```
Specification          Value
-----------------------------------
Particle Size          5 μm
Pore Size             100 Å
Column Length         250 mm
Inner Diameter        4.6 mm
pH Range              2-8
Max Pressure          400 bar
```

**转换为JSON**:
```json
{
  "粒径": "5 μm",
  "孔径": "100 Å",
  "柱长": "250 mm",
  "内径": "4.6 mm",
  "pH范围": "2-8",
  "最大压力": "400 bar"
}
```

## 🚨 特殊注意事项

### 1. 品牌名称统一

**Thermo Fisher产品**:
- ✅ 正确: "Thermo Fisher Scientific"
- ❌ 错误: "Thermo Fisher"

请统一使用完整品牌名称。

### 2. 产品匹配规则

使用以下规则匹配产品:

```
主键: productId
备用: brand + partNumber

示例:
productId = "AGIL-51832068"
或
brand = "Agilent" AND partNumber = "5183-2068"
```

### 3. 中文字段处理

如果品牌官网只有英文信息:
- **优先使用英文原文**
- 不要使用机器翻译
- 在报告中标注"仅英文"

如果品牌官网有中文信息:
- 优先使用中文
- 确保翻译准确
- 技术术语保持一致

### 4. 技术规格标准化

**统一单位格式**:
- 长度: "250 mm" (数字+空格+单位)
- 粒径: "5 μm" (使用μ符号)
- 孔径: "100 Å" (使用Å符号)
- 压力: "400 bar" 或 "6000 psi"
- 温度: "25°C" (使用°符号)

**统一字段命名**（建议使用中文）:
- "粒径" 而非 "Particle Size"
- "柱长" 而非 "Column Length"
- "内径" 而非 "Inner Diameter"

### 5. 错误处理

**产品未找到**:
- 在CSV中保留该行
- description字段填写: "NOT_FOUND"
- specifications字段填写: "{}"
- 在报告中列出

**信息不完整**:
- 尽可能填写已有信息
- 缺失字段留空或填写"N/A"
- 在报告中说明

**零件号错误**:
- 在报告中标记
- 建议正确的零件号（如果找到）

## ⚖️ 法律合规要求

### ✅ 允许的行为

1. **访问公开产品页面**
   - 这些是公开展示的商业信息
   - 用于建立产品数据库

2. **提取产品技术参数**
   - 产品规格是事实性信息
   - 不受版权保护

3. **提取产品描述**
   - 用于信息展示
   - 标注数据来源

### ❌ 禁止的行为

1. **不要复制营销文案**
   - 避免逐字复制创意性内容
   - 可以提取事实性描述

2. **不要下载品牌Logo**
   - 品牌Logo受商标保护

3. **不要绕过访问限制**
   - 遵守robots.txt
   - 不使用反爬虫绕过技术

4. **不要过度请求**
   - 控制请求频率（建议: 1-2秒/请求）
   - 单域名并发 ≤ 2

### 🛡️ 风险规避建议

1. **使用合理的User-Agent**
   ```
   Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36
   ```

2. **遵守robots.txt**
   - 检查各品牌网站的robots.txt
   - 避免爬取禁止的路径

3. **控制请求频率**
   - 每个请求间隔 1-2秒
   - 避免在短时间内大量请求

4. **数据使用声明**
   - 在网站标注数据来源
   - 明确"非授权经销商"身份
