## 📋 任务概述

### 任务目标

完成**366个Thermo Fisher Scientific品牌产品**的数据爬取，获取产品描述和技术规格信息。

### 品牌信息

- **品牌名称**: Thermo Fisher Scientific
- **官方网站**: https://www.thermofisher.com
- **产品类型**: HPLC色谱柱、样品瓶、配件等色谱耗材
- **目标市场**: 全球科研机构、制药企业、检测实验室

### 任务背景

- 这是继Agilent（630个产品，已完成）之后的第二个品牌
- Thermo Fisher Scientific是全球领先的科学仪器和耗材供应商

## 🎯 执行策略

### 两阶段执行方案

参考Agilent的成功经验，采用两阶段执行策略：

#### 阶段1：小批量测试（10-20个产品）

**目标**: 验证爬虫程序对Thermo Fisher官网的适配性

**执行步骤**:
1. 从产品清单中随机选择10-20个产品
2. 执行爬取测试
3. 验证数据质量
4. 调整爬虫程序（如需要）

**预计时间**: 15-25分钟

**质量标准**:
- 成功率 ≥ 80%
- 描述覆盖率 ≥ 60%
- 规格完整性 ≥ 80%

**决策点**:
- 如果测试通过 → 进入阶段2全量爬取
- 如果测试失败 → 分析原因，调整程序，重新测试

#### 阶段2：全量爬取（366个产品）

**目标**: 完成所有366个产品的数据爬取

**执行步骤**:
1. 使用测试通过的爬虫程序
2. 爬取全部366个产品
3. 生成爬取报告
4. 交付结果文件

**预计时间**: 2-3小时（取决于官网响应速度）

**质量标准**:
- 成功率 ≥ 90%（至少330个产品）
- 描述覆盖率 ≥ 70%（至少257个产品）
- A/B级描述占比 ≥ 30%（至少110个产品）
- 规格完整性 ≥ 90%（至少330个产品，≥3个字段）
- 平均规格字段数 ≥ 10个

## 🔧 技术要求

### 爬虫程序要求

1. **复用Agilent爬虫程序**
   - 使用已验证的爬虫框架
   - 调整品牌特定的CSS选择器
   - 适配Thermo Fisher官网结构

2. **官网适配**
   - 官网: https://www.thermofisher.com
   - 产品页面URL格式: 需要根据partNumber构建
   - 可能需要处理的问题:
     - 产品页面重定向
     - 动态加载内容
     - 反爬虫机制

3. **数据提取规则**
   - 产品名称: 从页面标题或H1标签提取
   - 产品描述: 从产品介绍区域提取，优先提取完整段落
   - 技术规格: 从规格表格或列表提取，转换为JSON格式
   - 规格字段名: 保持英文原文，不要翻译

4. **错误处理**
   - 404错误: 记录产品ID，标记为失败
   - 超时错误: 重试3次，仍失败则记录
   - 解析错误: 记录错误信息，继续下一个产品

5. **性能要求**
   - 爬取速度: 建议每个产品间隔1-2秒
   - 并发数: 建议1-3个并发请求
   - 总耗时: 预计2-3小时

### 合规要求

1. **遵守robots.txt**: 检查并遵守Thermo Fisher官网的robots.txt规则
2. **User-Agent**: 使用合理的User-Agent标识
3. **爬取频率**: 控制请求频率，避免对官网造成压力
4. **数据使用**: 仅用于ROWELL网站产品信息展示，不用于商业竞争

## 🚨 风险和应对

### 潜在风险

1. **官网结构变化**
   - 风险: Thermo Fisher官网结构可能与Agilent不同
   - 应对: 阶段1测试时及时发现并调整

2. **反爬虫机制**
   - 风险: 官网可能有反爬虫措施（验证码、IP限制）
   - 应对: 降低爬取速度，使用代理IP，模拟真实用户行为

3. **产品URL构建失败**
   - 风险: 无法根据partNumber构建正确的产品页面URL
   - 应对: 尝试多种URL格式，或通过搜索功能定位产品页面

4. **数据质量不达标**
   - 风险: 爬取的数据质量可能不符合标准
   - 应对: 优化数据提取规则，提高描述和规格的完整性

### 应急预案

- 如果成功率<70%，暂停爬取，分析原因，调整程序后重新执行
- 如果遇到IP封禁，更换IP或使用代理服务
- 如果官网结构复杂，考虑从经销商网站获取数据
- 如果时间严重超出预期，及时向ROWELL团队报告
